# 工作流编辑器规范（变更）

## MODIFIED Requirements

### Requirement: 工作流 Mock 执行

工作流编辑器 SHALL 支持模拟工作流执行，用于测试和验证流程逻辑，无需连接真实外部服务。

#### Scenario: 启动 Mock 执行（选择执行接口）

- **WHEN** 操作员在编辑器中点击"Mock 执行"按钮
- **THEN** 系统显示 Mock 执行配置面板
- **AND** 面板应包含：
  - 执行接口选择器（下拉菜单），当前选项：
    - `POST /api/execute/:workflowInstanceId`（执行工作流）
  - 参数输入区域：
    - `workflowInstanceId`: 可选，文本输入框（如果为空，系统自动创建 Mock 实例）
    - `fromNodeId`: 必填，文本输入框或节点选择器
    - `businessParams`: 可选，JSON 编辑器
  - Mock 数据配置区域：
    - 节点级别的 Mock 响应配置
    - 执行延迟配置
    - 失败场景配置
- **AND** 操作员可以点击"开始执行"按钮启动 Mock 执行

#### Scenario: Mock 执行可视化（关联流程图）

- **WHEN** Mock 执行进行中
- **THEN** 系统应在流程图上高亮显示当前执行节点（基于 `currentNodeIds`）
- **AND** 已执行的节点应显示完成状态标记
- **AND** 执行路径的连线应高亮显示
- **AND** Mock 执行控制面板应显示：
  - 当前执行节点 ID 列表
  - 当前执行状态
  - 工作流变量值
  - 执行历史记录

#### Scenario: 继续 Mock 执行（下一步）

- **WHEN** Mock 执行暂停或等待下一步时
- **THEN** 系统应显示"下一步"按钮
- **AND** 点击"下一步"按钮时，系统应显示参数输入对话框：
  - `businessParams`: 可选，JSON 编辑器（当前步骤的业务参数）
  - `nodeMockData`: 可选，节点 Mock 数据配置
- **AND** 操作员可以输入参数后点击"执行下一步"
- **AND** 系统应发送 `POST /api/workflows/mock/executions/:executionId/step` 请求
- **AND** 执行后，系统应更新流程图显示和 Mock 执行状态

#### Scenario: 查看 Mock 实例状态

- **WHEN** Mock 执行进行中
- **THEN** 系统应显示关联的 Mock 工作流实例信息
- **AND** 实例信息应包括：
  - 实例 ID
  - 实例状态
  - 当前节点 ID 列表
  - 工作流变量
  - 实例版本号
- **AND** 操作员可以点击"查看实例详情"查看完整的实例数据

## ADDED Requirements

### Requirement: Mock 执行接口选择

工作流编辑器 SHALL 支持选择要模拟的执行接口。

#### Scenario: 选择执行接口

- **WHEN** 操作员启动 Mock 执行
- **THEN** 系统应提供执行接口选择器
- **AND** 当前支持的接口：
  - `POST /api/execute/:workflowInstanceId`（执行工作流）
- **AND** 选择接口后，系统应显示该接口所需的参数输入字段
- **AND** 参数输入字段应根据接口定义动态生成

#### Scenario: Mock 实例管理

- **WHEN** 操作员启动 Mock 执行时未提供 `workflowInstanceId`
- **THEN** 系统应自动创建 Mock 工作流实例
- **AND** 创建的实例应显示在 Mock 执行控制面板中
- **AND** 操作员可以查看和管理 Mock 实例的状态

### Requirement: Mock 执行信息展示面板

工作流编辑器 SHALL 在 Mock 控制面板中提供结构化的执行信息展示面板。

#### Scenario: 显示执行信息 Tabs

- **WHEN** 操作员打开 Mock 控制面板
- **THEN** 系统应在"开始执行"按钮正下方显示 Ant Design Tabs 组件
- **AND** Tabs 应始终可见，不依赖是否有执行结果
- **AND** Tabs 应包含以下标签项：
  - "请求/回包"（默认选中）
  - "拦截器调用"
  - "日志"（TODO，暂时留白）
- **AND** Tabs 内容区域应占据面板的主要空间
- **AND** Tabs 应支持切换，保持用户选择的 Tab 状态
- **AND** 在未执行时，各 Tab 内容应显示空状态提示

#### Scenario: 展示和编辑请求和响应数据

- **WHEN** 操作员选择"请求/回包" Tab
- **THEN** 系统应显示上下两个代码输入框（可编辑）
- **AND** 上半部分应显示：
  - 标题："请求入参"
  - 内容：执行请求的入参（JSON 格式）
  - 数据从执行请求的 requestParams 转化而来
  - 如果未执行，显示空的代码输入框或提示
  - 支持 JSON 格式编辑
  - 支持代码高亮和格式化
- **AND** 下半部分应显示：
  - 标题："响应数据"
  - 内容：接口返回的回包数据（JSON 格式）
  - 数据从 businessResponse 和 engineResponse 转化而来
  - 如果未执行，显示空的代码输入框或提示
  - 支持 JSON 格式编辑
  - 支持代码高亮和格式化
- **AND** 两个区域应使用分隔条，支持拖拽调整高度
- **AND** 点击"开始执行"或"单步"后，数据应动态更新并填充到输入框

#### Scenario: 展示和编辑拦截器调用信息

- **WHEN** 操作员选择"拦截器调用" Tab
- **THEN** 系统应显示上下两个区域
- **AND** 上半部分应显示：
  - 标题："拦截器列表"
  - 内容：执行过程中调用的拦截器列表
  - 如果未执行或无拦截器调用，显示"暂无拦截器调用记录"
  - 每个拦截器项显示：
    - 拦截器名称
    - 调用顺序（序号）
    - 调用时间（可选）
  - 支持点击选择拦截器
  - 已选中的拦截器应高亮显示
- **AND** 下半部分应显示：
  - 标题："拦截器详情"
  - 如果未选择拦截器，显示"请选择拦截器查看详情"
  - 当选中某个拦截器时，显示 Tabs 组件：
    - "入参" Tab：显示和编辑拦截器函数的入参（JSON 格式代码输入框）
    - "回包" Tab：显示和编辑拦截器函数的出参（JSON 格式代码输入框）
  - 支持 JSON 格式编辑
  - 支持代码高亮和格式化
- **AND** 两个区域应使用分隔条，支持拖拽调整高度
- **AND** 点击"开始执行"或"单步"后，拦截器列表应动态更新

#### Scenario: 展示执行日志

- **WHEN** 操作员选择"日志" Tab
- **THEN** 系统应显示占位区域
- **AND** 占位区域应显示提示文本："日志功能开发中，敬请期待"
- **AND** 占位区域应预留日志展示的空间布局

### Requirement: Mock 执行接口选择器位置

工作流编辑器 SHALL 在 Mock 控制面板的"开始执行"按钮上方提供接口选择器。

#### Scenario: 显示接口选择器

- **WHEN** 操作员打开 Mock 控制面板
- **THEN** 系统应在"开始执行"按钮上方显示接口选择器
- **AND** 接口选择器应为下拉菜单组件
- **AND** 选择器应显示当前选中的接口
- **AND** 当前支持的接口选项：
  - `POST /api/execute/:workflowInstanceId`（默认选中）
- **AND** 选择器应包含标签："执行接口"
- **AND** 选择器应占据一行，宽度适应内容
